# 创作指南与最佳实践

> 最后更新: 2026-04-24
> 适用项目: 这个历史游戏，只有我知道结局

## 一、本书定位

**一句话说清楚**：
平行世界智脑统治下，黑铁民穿越者凭地球历史知识在历史模拟副本中碾压付费玩家。

**核心卖点**：
1. 信息差碾压——别人在盲选，他提前知道结局
2. 属性面板不重置——亭长外表，猛将实力
3. 游戏币经济——看广告攒钱、选角色花钱、改历史赚成就点
4. 意难平副本——项羽乌江自刎、岳飞风波亭，亲眼见证比课本痛一百倍

**目标读者**：番茄小说用户，偏好轻松爽文+历史元素
**赛道**：山河异卷（历史脑洞）
**基调**：爽文为主，历史沉浸为辅，意难平情感爆发为高潮

---

## 二、双世界切换原则

### 核心原则
本书有两个世界：**现实世界**（智脑天枢统治）和**副本世界**（历史模拟）。切换必须自然。

### 现实世界写作要点
- **功能定位**：升级+过渡+伏笔+准备下一副本
- **篇幅控制**：每个副本之间3-5章现实线，不超过1万字
- **核心内容**：结算奖励→升级设备/住房→与其他玩家互动→天枢伏笔
- **避免**：大段描写现实世界的日常（读者来看历史副本的，不是来看科幻的）
- **正确做法**：现实世界的信息通过主角的行动自然带出（买头盔时展示黑铁民区、升级时展示等级差异）

### 副本世界写作要点
- **功能定位**：主战场，爽点和情感爆发的地方
- **篇幅控制**：每个副本20-25万字（征文期写完第一个副本）
- **核心内容**：历史事件+玩家博弈+信息差碾压+意难平
- **避免**：照搬历史教科书（读者知道的历史一笔带过，重点写"亲历"的感受）
- **正确做法**：用主角的"地球人视角"重新诠释历史事件

### 切换方式
- 进入副本：通过游戏头盔/游戏仓，意识进入，身体留在现实
- 退出副本：副本结算后意识回到现实
- **禁止**：在副本内突然切回现实（除非副本结束或角色死亡）
- **正确做法**：副本内偶尔闪回现实记忆（如想起黑铁民区的艰辛，强化战斗动机）

## 三、信息差呈现方式

### 核心原则
主角的核心优势是"知道历史走向"，但**不能直接告诉读者历史课本内容**。

### 正确做法
- **通过行动展示**：主角做出正确选择，结果验证了他的判断
- **通过内心独白暗示**：主角知道接下来会发生什么，但不能说
- **通过对比制造爽感**：其他玩家选了"最强势力"然后输了，主角选了"最弱势力"然后赢了

### 错误做法
- ❌ 大段解释历史背景（"公元前209年，秦始皇驾崩，赵高矫诏..."）
- ❌ 主角直接告诉NPC未来会发生什么（会被当成疯子）
- ❌ 用旁白解释"因为历史上刘邦赢了，所以主角选了刘邦"

### 示例
- ❌ 错误：主角心想："历史上鸿门宴项庄会刺杀刘邦，我必须阻止。"
- ✅ 正确：主角看着项庄的剑越舞越近，手心全是汗。他安排樊哙在帐外待命——因为他知道，接下来会发生什么。

---

## 四、玩家博弈写作规范

### 核心原则
副本内有多个玩家，玩家之间的博弈是重要的爽点来源。

### 玩家视角切换
- 偶尔切换到其他玩家的视角（1-2段即可），展示他们的选择和后果
- **目的**：衬托主角的正确选择
- **频率**：每5-10章切换一次，不超过500字
- **禁止**：花一整章写其他玩家的故事

### 博弈类型
1. **信息差博弈**：主角知道结局，其他玩家不知道→主角选对了，他们选错了
2. **实力博弈**：付费玩家角色强但走错方向，主角角色弱但走对方向
3. **锚点博弈**：有人想保护锚点，有人想破坏锚点

## 五、意难平写作规范

### 核心原则
意难平是本书的情感高潮，必须让读者感同身受。

### 写法要点
- **不要直接说"意难平"**，通过主角的亲历感受传达
- **铺垫要充分**：韩信/岳飞等人物要先写好、写立体，读者对他们有感情了，他们的死亡才有冲击力
- **主角的无力感**：他知道结局但无法改变（或者改变代价太大），这种无力感比直接写"悲惨"更痛
- **留白**：意难平场景不要写太多心理描写，用行动和沉默传达

### 示例（韩信之死）
- ❌ 错误：韩信被杀的那一刻，主角心中充满了悲痛和愤怒，他觉得这不公平...
- ✅ 正确：长乐宫的门关上了。主角站在宫墙外面，听着里面传来的动静。他什么都没说。他三个月前就告诉过韩信，陛下会杀你。韩信笑了，说陛下不会的。

## 六、属性面板与游戏币呈现

### 核心原则
属性面板和游戏币是重要的世界观元素，但不能变成数据堆砌。

### 呈现方式
- **属性面板**：只在关键时刻展示（升级时、战斗前对比时、碾压对手时）
- **游戏币**：通过主角的决策展示（"120个币买了头盔，剩7个"比"他拥有127游戏币"更有画面感）
- **成就点**：触发成就时用系统提示框展示（简短一行即可）

### 避免
- ❌ 每章都列出属性面板数据
- ❌ 大段解释游戏币经济系统（通过主角的行动自然展示）
- ❌ 属性数字精确到小数点（用整数+等级描述即可）

## 七、语言风格

### 核心原则
- **口语化**：主角是现代人，内心独白用现代口语
- **不文绉绉**：副本内的NPC用符合时代的语言，但不要过度文言文
- **短句为主**：番茄读者习惯快节奏，每段3-5行
- **不堆砌形容词**：用一个精确的动词比三个形容词有效
- **避免网络流行语**：不要用"yyds""绝绝子"等（会出戏）

### 对话风格
- 主角：现代口语，偶尔带历史系学生的书卷气
- 刘邦：豁达粗犷，偶尔爆粗
- 项羽：简洁有力，命令式
- 韩信：自信但带点傲气
- 其他玩家：现代口语（他们也是现代人穿越进副本的）

### 玩家身份隔离规则（重要！）

**核心原则**：NPC不能知道"玩家""游戏币""穿越者"等游戏概念的存在。玩家之间的游戏信息只能通过铜环（世界频道/私聊）交流，绝对不能当面在NPC面前说。

#### 1. 什么信息NPC能听到
- ✅ 角色身份（刘邦、萧何、雍齿等）
- ✅ 历史/政治话题（起义、征兵、天下大势）
- ✅ 任何符合秦末世界观的内容

#### 2. 什么信息NPC不能听到
- ❌ "玩家""游戏币""穿越者""副本""成就"等游戏术语
- ❌ "穿越者""地球""历史"等打破第四面墙的词
- ❌ 任何通过铜环私聊/世界频道传递的信息
- ❌ 玩家编号（如"查#33071是谁"——NPC不知道什么是编号）

#### 3. 玩家之间如何交流
- **当面**：只能说NPC身份该说的话，用"暗号"暗示对方是玩家
- **铜环私聊**：可以自由讨论游戏信息，只有双方能看到
- **世界频道**：所有玩家可见，NPC看不到
- **阵营频道**：同阵营玩家可见，NPC看不到

#### 4. 铜环消息格式规范（重要！）
所有铜环消息必须使用 `昵称（角色名）` 格式。已知角色名的玩家必须带角色名，未知角色名的玩家只用昵称。

**世界频道**：`王侯将相宁有种乎（陈胜）：是我选的。`
**阵营频道**：`我是皇帝（秦二世）：有没有人能帮我？`
**私聊**：`背主求荣（雍齿）→逆流（刘邦）：你猜我为什么选丰邑守将？`
**成就通知**：`「触发者：逆流（刘邦）。奖励：2000游戏币。」`
**选择消息**：`逆流（刘邦）选择了「泗水亭长·刘邦」！`

**昵称对应关系（权威参考：dungeon1-characters.md）**：
| 昵称 | 角色 |
|------|------|
| 逆流 | 刘邦（主角） |
| 我是皇帝 | 秦二世（胡亥） |
| 指鹿为马 | 赵高 |
| 楚虽三户 | 项梁 |
| 壮士一去不复还 | 项庄 |
| 王侯将相宁有种乎 | 陈胜 |
| 咸阳夜话 | 关中世族 |
| 背主求荣 | 雍齿 |
| 日进斗金 | 乱世商人 |

**注意**：不使用编号（如#21784），全部用昵称。编号已废弃。

#### 5. 铜环消息的内容限制
铜环上的消息虽然NPC看不到，但其他玩家能看到。因此：
- ❌ 不要在私聊/世界频道中暴露自己的等级信息（如"青铜民""黑铁民"），否则其他玩家可以通过等级定位到你
- ✅ 铜环消息只讨论角色相关的事（选了谁、在做什么、下一步计划），不说系统层面的信息

#### 6. 常见错误示例
- ❌ 雍齿在城头上喊："你花了一百游戏币选了个亭长？"（NPC守军全听到了）
- ✅ 雍齿在城头上喊："刘邦？丰邑人？" + 铜环私聊："一百游戏币选了个亭长？"（NPC只听到前一句）
- ❌ 陆行舟对萧何说："查查逆流是谁"（NPC不知道什么是昵称）
- ✅ 陆行舟对萧何说："查查这个人在背后到底是谁" + 内心独白想"日进斗金在给雍齿送钱"（NPC只听到前一句）

### 对话情感温度（重要！）

**核心问题**：对话不能只是"交换信息"，必须有情感温度。读者读对话时应该能感受到人物之间的关系、情绪和态度。

#### 1. 情感对话三原则
- **每句对话背后都有一个情绪**：即使是讨论正事，人物也在带着情绪说话。萧何说"大事你听我的"时，他的情绪是"自信+试探"；樊哙说"行"时，他的情绪是"信任+兴奋"
- **关系亲疏决定对话方式**：亲密的人说话短、随意、有默契；疏远的人说话长、客气、有距离。樊哙对陆行舟说话不用敬语，萧何对陆行舟说话带着商量
- **情绪要有变化**：同一场对话中，人物的情绪应该有起伏。不能从头到尾都是同一个调子

#### 2. 让对话有温度的5个技巧

**技巧一：用动作打断对话**
不要让两个人干巴巴地你一句我一句。在对话中间插入动作、表情、环境反应，让对话"呼吸"。
- ❌ "你觉得呢？""我觉得不行。""为什么？""因为太危险了。"
- ✅ "你觉得呢？"萧何放下筷子，看了他一眼。"我觉得不行。"他顿了一下，端起碗喝了一口水，才继续说，"太危险了。"

**技巧二：潜台词比台词更重要**
人物嘴上说的和心里想的不一样时，对话才有张力。
- ❌ "我有点担心。""别担心，没事的。"
- ✅ "你不觉得太快了吗？"萧何没看他，手指在桌面上轻轻敲着。"快？"陆行舟笑了笑，"再慢就来不及了。"萧何的手指停了。

**技巧三：让环境参与对话**
用环境变化反映人物情绪。风、雨、光线、声音都可以成为对话的"第三个参与者"。
- ❌ "我同意。""好，那就这么定了。"
- ✅ "我同意。"窗外刮过一阵风，油灯的火苗晃了一下。"好。"陆行舟看着那团跳动的火苗，"那就这么定了。"

**技巧四：省略比说全更有力**
有些话不用说完整，读者自己能补上。留白比填满更有情感冲击力。
- ❌ "你真的要这么做吗？你知不知道这有多危险？你可能会死的！"
- ✅ "你真的要这么做？"樊哙看着他。陆行舟没回答。樊哙攥紧了拳头，没再说话。

**技巧五：给对话加"前戏"和"余韵"**
重要对话不要一上来就切入正题。先铺垫情绪，说完之后留一点回味。
- ❌ "我们起兵吧。""好。"
- ✅ 樊哙把碗里的酒一口闷了，抹了抹嘴。"大哥，我跟着你干。"他站起来，椅子在地上划出一声刺耳的响。"不管去哪。"

#### 3. 对话情感自查
写完一段对话后，问自己三个问题：
1. 删掉这段对话，读者还能理解剧情吗？——如果能，说明对话只是"信息传递"，没有情感价值
2. 把这段对话的台词换成别的词，人物关系会变吗？——如果不会，说明对话没有"人物专属感"
3. 读这段对话时，我能感受到人物的情绪吗？——如果感受不到，说明对话太"干"了

## 八、章节结构准则

### 章节标题规范
- **长度**：10-25字，太短没信息量（如"萧何""雍齿"），太长像书名
- **核心公式**：核心事件/悬念 + 口语化表达
- **必须包含的信息**：让读者看到标题就想点进去
- **禁止**：纯人名（如"萧何""雍齿"）、纯动作（如"去丰邑"）、过于文艺、剧透结局
- **不可重复**：同一本书中所有章节标题必须唯一，高度相似的标题也不允许（如"萧何"和"萧何的对答"核心词重复，应避免）
- **推荐风格**：
  - 口语化长句：「攒了三个月广告费我买了个头盔」「全员选了大秦帝国就我翻到了秦国官吏」
  - 悬念短句：「他手下只有七个人」「大泽乡的雨夜」
  - 对比冲突：「选了秦二世的那个钻石民已经开始慌了」「一封从关东寄回来的信」
  - 关键台词：「那八个人」「刘邦是来抢地盘的」
- **自检**：标题删掉后，读者还能从目录中感受到剧情推进吗？如果能，说明标题不够有吸引力
- **详细规范**：参见 `docs/writing-guide/tips/11-标题与起名.md`

### 开头（≤200字）
- 承接上章，快速进入场景
- 用动作或对话切入，不要用环境描写

### 发展（主体）
- 每章至少一个核心事件
- 每2000字至少一个小爽点或情绪转折
- 对话60%/旁白30%/心理10%

### 结尾（≤100字）
- 必须留钩子：悬念/反转/新信息/情绪余韵
- 不要把事件完整讲完

## 九、剧情不平淡的核心法则

### 核心诊断
**剧情平淡的根本原因**：只写了"发生了什么"，没写"人物因此产生了什么情绪"。读者关心的不是事件本身，而是事件对人物造成的冲击。

**一句话总结**：写情绪，不要写事件。

### 1. 情绪传递递进法则
故事发展 = 情绪传递和发泄的过程。每个事件都必须完成一次"情绪传递"：
```
事件发生 → 人物受到冲击 → 表现出情绪 → 情绪传递给周围人 → 周围人做出反应 → 新的情绪产生
```
- ❌ 平淡写法：萧何加入了。陆行舟很高兴。他们开始计划起兵。
- ✅ 情绪写法：萧何走进泗水亭的那一刻，樊哙手里的碗差点掉了。他跟了陆行舟这么多年，第一次看到有人主动找上门来。陆行舟没说话，但他端碗的手指微微收紧了一下。

### 2. 剧情张力三要素
每段剧情必须包含以下至少两个：
- **冲突**：人物之间的对立、矛盾、分歧
- **代价**：做这件事需要付出什么、失去什么、承担什么风险
- **意外**：出乎读者和角色预料的变化

三个都有 = 高张力
两个 = 正常
一个或没有 = 平淡

**自检方法**：写完一段剧情后问自己——"如果删掉这段，读者会觉得少了什么吗？"如果答案是"什么都没少"，这段就是平淡的。

### 3. 压-缓解-爆发的节奏模板
```
压（制造压抑/危机感）
→ 缓解（给一点希望或温情，让读者喘口气）
→ 再压（更大的危机出现）
→ 希望（转机出现）
→ 强爆发（极度压抑后的释放）
→ 新钩子（引出后续）
```

**关键**：不能一直压不给喘息，也不能一直平没有起伏。每2000字至少有一次情绪转折。

### 4. 锁链式剧情推进
不要用 A→B→C→D 的排队式串联。要让下一个事件的开头嵌入到上一个事件的过程中：
- ❌ 排队式：起兵成功 → 打丰邑 → 丰邑叛变 → 收复丰邑
- ✅ 锁链式：起兵成功的同时，丰邑那边已经在调兵了 → 打丰邑时发现守将是玩家 → 攻城过程中丰邑内部有人想叛变

### 5. 让读者"心疼"的技巧
读者对角色的感情 = 角色的痛苦 × 角色的努力。越努力越痛苦，读者越心疼。
- 不要让主角太顺利。每次成功之后都要付出代价
- 不要让配角太工具人。配角也要有自己的挣扎和选择
- 用"失去"制造情感冲击。失去比得到更让人记住

### 6. 剧情平淡自查清单
写完一章后，问自己：
1. **这一章里，人物的情绪有变化吗？**（如果从头到尾都是同一个情绪，就平淡了）
2. **这一章里，有没有让读者意外的地方？**（如果读者能猜到所有发展，就平淡了）
3. **这一章里，有没有人付出了代价？**（如果所有事情都太顺利，就平淡了）
4. **删掉这一章，后面的故事还能看懂吗？**（如果能，说明这章没有不可替代的内容，就平淡了）
5. **读这一章时，我有"想往下翻"的冲动吗？**（如果没有，说明钩子和张力不够）

## 十、检查清单

### 每章完成前检查
- [ ] 信息差通过行动展示，而非直接告知
- [ ] 没有大段历史背景堆砌
- [ ] 属性面板/游戏币没有过度展示
- [ ] 现实世界和副本世界的切换自然
- [ ] 意难平场景有充分铺垫
- [ ] 对话符合角色性格
- [ ] 对话有情感温度（不是冷冰冰的信息交换）
- [ ] 重要对话有动作/表情/环境穿插（对话有"呼吸感"）
- [ ] 人物情绪有变化（不是从头到尾一个调子）
- [ ] 剧情有张力（至少包含冲突/代价/意外中的两个）
- [ ] 有人付出了代价（不是所有事都太顺利）
- [ ] 事件之间是锁链式推进（不是排队式串联）
- [ ] 章末有钩子
- [ ] 字数在2000-3000字范围
- [ ] 没有元叙事表述（跳出角色的词）
- [ ] 排版规范（每段3-5行）
